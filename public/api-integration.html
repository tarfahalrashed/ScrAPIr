Logging Consent

To support our research, we want to better understand how people use Scrapir. To do that, we would like to log the queries people perform and the flow people go through to integrate a web API. So, if you accept, we will log the sites you query, the queries you make, the returned results, and the different steps you take to integrate any additional APIs. We will not log or save any other information about your identity.

This research is being carried out by the MIT Haystack Group, under a protocol that has been approved by MIT's Committee on the Use of Humans as Experimental Subjects (COUHES).

5. Pagination

Pagination Information

Demo Details

On some sites, a user query might match millions of records. Thus, APIs paginate the results to make sure responses are easier to handle. Some APIs posit a fixed or specifyable number of items per page, here you need to provide max number of results per page.

Please choose the pagination style for this API:

Pagination in websites generally offer "next" and "previous" page links, and sometimes a box for jumping to a specific page number. Similarly, some APIs use an "index" parameter to specify the page number or an "offset" into the entire result list. Others provide "next" and "prev" that can be used to assign the "current" page parameter to request the adjacent subsequence of items.







Pagination Demo

This demonstrates how to add the pagination information. As an example, we are showing how to do that with YouTube Search API's.


Pagination Explanation

This example is using Yelp Business Search API: In Yelp business search API, the number of business results to return per page is between 1 and 50 (max number of results per page is 50). The parameter used for number of results per page is limit (limit = 50). The way that Yelp search API goes over the pages is using offset, and the name of the parameter it uses is offset.
